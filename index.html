Rust-inspired scripting for portable runtimes

Typed automation that can travel with your app.

RustScript pairs a compact Rust-like language with a portable bytecode VM, host-function bindings, debugger hooks, and browser tooling. It is built for teams that need small scripts to run safely inside controllers, edge nodes, browsers, games, gateways, and native host applications.

fn first_or<T>(items: [T], fallback: T) -> T {
  if items.length > 0 => {
    items[0]
  } else => {
    fallback
  }
}

fn score(route: string, samples: [int]) -> int {
  let mut base = 40;
  let latency = first_or::<int>(samples, 120);
  if route == "edge" {
    base = 80;
  }
  base - latency / 10
}

Design ideas

Small language, explicit runtime boundary.

RustScript keeps scripts readable for Rust users while making host integration clear. Scripts describe control flow and data movement; hosts own privileged operations through registered bindings.

Typed by default

Use explicit signatures and predictable values when scripts cross process, VM, WASM, or host boundaries.

Portable VM core

Compile to compact bytecode that can be embedded in services, UI apps, games, and edge agents.

Host-controlled power

Expose only the functions the host chooses. Scripts remain focused on policy, glue, and automation.

Cooperative scheduling and yielding

Fuel bounds VM work, Epoch gives hosts wall-clock deadlines, and async host calls can yield or wait at explicit boundaries.

JIT and AOT

Start quickly in the interpreter, trace hot paths through the JIT, or lower whole programs through SSA and Cranelift for AOT artifacts.

No garbage collector

Rust-style ownership, borrows, and moves keep lifetimes visible to the compiler; shared values release through normal Drop without tracing GC.
